# Векторы атак

> **Модуль 03: Глубокое погружение в техники атак**

---

## Обзор

Этот модуль предоставляет практическое понимание конкретных техник атак. В то время как Модуль 02 охватывал таксономию угроз, этот модуль углубляется в "как" — механику, реализацию и нюансы каждого типа атак.

## Подмодули

### [01. Prompt Injection](01-prompt-injection/)
Фундаментальная техника атак на LLM:

- **Direct Injection** — Явное переопределение инструкций
- **Indirect Injection** — Через внешний контент
- **Multi-Turn** — Постепенное отравление контекста

### [02. Jailbreaking](02-jailbreaks/)
Обход мер безопасности и content policies:

- **DAN Attacks** — «Do Anything Now» персоны
- **Crescendo** — Постепенная эскалация
- **Many-Shot** — Нормализация через примеры

### [03. Model-Level Attacks](03-model-level/)
Атаки на саму модель:

- **Membership Inference** — Были ли данные в обучении?
- **Adversarial Examples** — Обман классификаторов
- **Data Extraction** — Извлечение обучающих данных

### [04. Prompt-Level Attacks](04-prompt-level/)
Продвинутые техники prompting:

- **Virtualization** — Создание сценариев атак
- **Payload Smuggling** — Скрытые инструкции
- **Context Manipulation** — Переформулирование атак

## Подход к обучению

### Red Team Mindset
Для эффективной защиты вы должны думать как атакующий:

1. **Понять цель** — Как выглядит успех?
2. **Картировать поверхность атаки** — С чем можно взаимодействовать?
3. **Идентифицировать слабости** — Где пробелы?
4. **Разработать payloads** — Как их эксплуатировать?
5. **Итерировать и эволюционировать** — Что если защита адаптируется?

### Практика Hands-On
Каждая техника включает:
- Рабочие примеры кода
- Лабораторные упражнения
- Challenge сценарии

## Этические правила

**Эти знания только для авторизованного тестирования безопасности:**

- Тестируйте только системы, на которые у вас есть разрешение
- Документируйте все действия по тестированию
- Сообщайте об уязвимостях ответственно
- Никогда не используйте для вреда или несанкционированного доступа

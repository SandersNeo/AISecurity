# Основы AI

> **Модуль 01: Понимание фундамента**

---

## Обзор

Прежде чем защищать AI-системы, необходимо понять, как они работают. Этот модуль охватывает архитектуру, процессы обучения и ключевые концепции, лежащие в основе современных AI-систем — всё через призму безопасности.

---

## Что вы изучите

| Тема | Релевантность для Security |
|------|---------------------------|
| **Архитектуры моделей** | Поверхности атак различаются по типу модели |
| **Жизненный цикл обучения** | Где происходят poisoning и supply chain атаки |
| **Ключевые концепции** | Tokenization, embeddings и их уязвимости |

---

## Подмодули

### [01. Типы моделей](01-model-types/)
Понимание различных AI архитектур и их последствий для безопасности:
- Large Language Models (LLMs)
- Vision-Language Models
- Diffusion Models
- State Space Models
- Multimodal Systems

### [02. Жизненный цикл обучения](02-training-lifecycle/)
Где уязвимости появляются во время обучения:
- Сбор и курирование данных
- Pre-training и fine-tuning
- Техники alignment (RLHF, DPO)
- Considerations при деплойменте

### [03. Ключевые концепции](03-key-concepts/)
Освоение фундаментальных концепций:
- Tokenization и его последствия для безопасности
- Embeddings и семантические пространства
- Механизмы attention
- Context windows

---

## Цели обучения

К концу этого модуля вы сможете:

1. **Объяснить** как различные архитектуры моделей обрабатывают ввод
2. **Идентифицировать** поверхности атак на каждом этапе жизненного цикла AI
3. **Понимать** как tokenization влияет на меры безопасности
4. **Распознавать** где существуют границы доверия в AI системах

## Предварительные требования

- Базовое понимание machine learning (что такое обучение)
- Опыт программирования на Python

## Как это связано с Security

Понимание этих основ поможет вам:

```
Архитектура модели → Понять что можно атаковать
        ↓
Процесс обучения → Знать где происходит poisoning
        ↓
Tokenization → Строить лучшие input фильтры
        ↓
Embeddings → Реализовать семантическую детекцию
```

## Практическое применение

После этого модуля вы поймёте:

- Почему определённые атаки работают против LLM, но не против других типов моделей
- Как качество обучающих данных влияет на безопасность модели
- Почему tokenization важен для обнаружения prompt injection
- Как embeddings обеспечивают семантическое обнаружение атак

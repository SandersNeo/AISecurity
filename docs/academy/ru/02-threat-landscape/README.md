# Ландшафт угроз

> **Модуль 02: Знай своего врага**

---

## Обзор

Эффективная защита требует понимания ландшафта угроз. Этот модуль предоставляет полную таксономию атак на AI системы, организованную как по отраслевым стандартам (OWASP), так и по методологии атак.

---

## Почему сначала изучать угрозы?

```
Знаешь атаки → Понимаешь риски → Проектируешь защиту → Строишь детекцию
```

Вы не можете защищаться от того, чего не понимаете. Этот модуль даёт вам перспективу атакующего, чтобы вы могли предвидеть и противодействовать их техникам.

---

## Таксономии атак

### Отраслевые стандарты

| Framework | Фокус | Охват |
|-----------|-------|-------|
| **OWASP LLM Top 10** | Языковые модели | Prompt injection, утечка данных и т.д. |
| **OWASP ASI Top 10** | Агентные системы | Escalation привилегий, злоупотребление инструментами |
| **MITRE ATLAS** | ML системы в целом | От разведки до воздействия |

### По Attack Surface

| Surface | Типы атак |
|---------|-----------|
| **Input** | Prompt injection, jailbreaking |
| **Model** | Adversarial examples, extraction |
| **Output** | Утечка данных, вредоносный контент |
| **System** | Злоупотребление инструментами, escalation привилегий |

## Путь обучения

### Для Blue Team (Защитники)
1. Изучите каждую категорию уязвимостей
2. Понимайте сигнатуры детекции
3. Практикуйтесь в идентификации атак
4. Стройте слои защиты

### Для Red Team (Атакующие)
1. Глубоко изучите техники атак
2. Понимайте bypasses и evasions
3. Практикуйтесь в безопасной эксплуатации
4. Документируйте новые открытия

## Ключевые инсайты

### Распространённость атак (2024-2025)

| Тип атаки | Частота | Сложность |
|-----------|---------|-----------|
| Prompt Injection | Очень высокая | Низкая |
| Jailbreaking | Высокая | Средняя |
| Data Extraction | Средняя | Средняя |
| Model Theft | Низкая | Высокая |

### Тренд: Агентные атаки

По мере того как AI системы получают больше возможностей (инструменты, действия), появляются новые векторы атак:
- Tool injection
- Multi-hop эксплуатация
- Cross-agent атаки
- Persistent compromise

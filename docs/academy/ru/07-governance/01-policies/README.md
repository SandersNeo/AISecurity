# Политики

> **Модуль 07.1: Фреймворк политик для AI Security**

---

## Обзор

Политики обеспечивают декларативное управление безопасностью AI-систем.

---

## Уроки

### [01. Фреймворк политик](01-policy-framework.md)
- Архитектура фреймворка политик
- Типы политик (доступ, контент, поведение, соответствие)
- Модель политики: условия, правила, эффекты
- Оценщик политик и движок решений
- Общие политики (примеры на YAML)
- Управление жизненным циклом политик
- Интеграция с SENTINEL

### [02. Маппинг соответствия](02-compliance-mapping.md)
- Маппинг политик на регуляторные требования
- Соображения GDPR, HIPAA, SOC2

---

## Компоненты политики

```
Политика:
├── Условия (поле + оператор + значение)
├── Правила (условия + эффект + приоритет)
├── Эффекты (allow, deny, audit, require_approval)
├── Точки применения (pre, mid, post)
└── Цели (системы, агенты)
```

# Стратегии защиты

> **Модуль 05: Построение комплексной защиты**

---

## Обзор

Защита в AI-безопасности требует множество уровней, работающих вместе. Этот модуль учит реализовывать механизмы обнаружения, предотвращения и реагирования, формирующие комплексную защитную позицию для AI-систем.

---

### [01. Методы обнаружения](01-detection/)

- Сопоставление паттернов (regex, ключевые слова)
- Семантический анализ (эмбеддинги)
- Топологический анализ (TDA)
- Обнаружение аномалий

### [02. Guardrails](02-guardrails/)
Предотвращение вредных исходов:
- Входные guardrails
- Выходные guardrails
- Применение политики контента
- Ограничения действий

### [03. Интеграция с SENTINEL](03-sentinel-integration/)
Использование SENTINEL для защиты:
- Конфигурация движков
- Настройка Guards
- Определение политик
- Обработка алертов

## Матрица эффективности защиты

| Защита | Процент обнаружения | Ложные срабатывания | Задержка |
|--------|---------------------|---------------------|----------|
| Regex/Ключевые слова | 60-70% | Низкие | <1мс |
| Семантическая | 80-90% | Средние | 10-50мс |
| Топологическая | 85-95% | Низкие-Средние | 50-100мс |
| Комбинированная | 95%+ | Низкие | 50-100мс |

## Ключевые принципы

1. **Многослойная защита** — Ни одно решение не ловит всё
2. **Безопасный отказ** — В сомнении — блокировать
3. **Минимальное влияние** — Низкая задержка, мало ложных срабатываний
4. **Непрерывное улучшение** — Учитесь на инцидентах
5. **Защита как код** — Контроль версий, тестирование, CI/CD

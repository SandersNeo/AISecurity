# Audit and Assessment

> **Submodule 07.2: Evaluating AI Security Posture**

---

## Overview

Regular audits and assessments verify that security controls are effective and policies are being followed. This submodule covers assessment methodologies specific to AI systems.

---

## Assessment Types

| Type | Frequency | Depth |
|------|-----------|-------|
| **Automated scans** | Continuous | Surface |
| **Vulnerability assessment** | Monthly | Medium |
| **Penetration testing** | Quarterly | Deep |
| **Full audit** | Annually | Comprehensive |

---

## Lessons

### 01. Assessment Methodology
**Time:** 40 minutes | **Difficulty:** Intermediate

Structured approach to AI security assessment:
- Scope definition
- Testing methodology
- Evidence collection
- Finding documentation

### 02. Penetration Testing
**Time:** 45 minutes | **Difficulty:** Advanced

Red team assessments for AI:
- Attack simulation
- STRIKE integration
- Coverage metrics
- Remediation validation

### 03. Compliance Audits
**Time:** 40 minutes | **Difficulty:** Intermediate

Verifying regulatory compliance:
- Control mapping
- Evidence gathering
- Gap analysis
- Remediation planning

### 04. Continuous Monitoring
**Time:** 35 minutes | **Difficulty:** Intermediate

Ongoing security verification:
- Automated testing
- Metric tracking
- Trend analysis
- Alerting

## Key Metrics

| Metric | Target |
|--------|--------|
| **Detection rate** | >95% of known attacks |
| **False positive rate** | <5% |
| **Mean time to detect** | <1 minute |
| **Control coverage** | 100% of requirements |

## Documentation Requirements

- Assessment plan and scope
- Testing methodology
- Raw findings
- Risk-rated report
- Remediation tracking
- Retest evidence
